SMS Spam Detection

A Machine Learning Approach to Message Classification

View GitHub Repository

1. Project Overview

The proliferation of mobile phones has led to a corresponding increase in unsolicited and malicious messages, commonly known as SMS spam. This project aims to build a robust machine learning classifier capable of accurately distinguishing between legitimate messages ("ham") and fraudulent or unwanted messages ("spam"). By deploying effective Natural Language Processing (NLP) techniques, we can proactively filter unwanted content and enhance user safety.

2. Data & Feature Engineering

Dataset and Preprocessing

The model was trained on the publicly available **SMS Spam Collection Dataset**, which contains $\approx 5,572$ labeled messages. A critical step involved rigorous text preprocessing, including tokenization, removal of stop words, punctuation, and converting all text to lowercase to ensure consistency.

Vectorization (Feature Extraction)

To convert the raw text data into a format suitable for machine learning algorithms, we employed the **TF-IDF (Term Frequency-Inverse Document Frequency)** vectorizer. This technique assigns a weight to each word based on how frequently it appears in a message, balanced by how rare it is across the entire corpus, effectively capturing the importance of characteristic spam words.

3. Model Performance

Classification Model: Multinomial Naive Bayes

The **Multinomial Naive Bayes** classifier was selected for its high efficiency and strong performance in text classification tasks, particularly when dealing with TF-IDF features. Its probabilistic nature makes it ideal for handling large, sparse feature sets common in NLP.
